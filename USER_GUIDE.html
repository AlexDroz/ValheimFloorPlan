ValheimFloorPlan - User Guide

Place a pre-made .vfp floor plan in game, preview the footprint, align it precisely, and let the mod level terrain and spawn pieces automatically.

Preview First Auto Terrain Prep One-Key Undo

Quick Start

  1. Set FloorPlanFile to your .vfp file in config.
  2. Enter your world and press F8 to open preview.
  3. Move/rotate preview until placement looks right.
  4. Press E (default confirm key) to confirm build.
  5. Use F9 to undo if needed.

Installation

Thunderstore Mod Manager

  1. Install ValheimFloorPlan in Thunderstore Mod Manager.
  2. Launch the game once.
  3. Close game and update config if needed.

Manual

  1. Install BepInEx for Valheim.
  2. Copy the mod DLL to BepInEx/plugins.
  3. Launch once to generate config.

Preview Input Settings

  • MoveForwardKey / MoveBackwardKey / MoveLeftKey / MoveRightKey (defaults: arrow keys)
  • RotateLeftKey / RotateRightKey (defaults: Q / R)
  • ConfirmKey (default E)
  • CancelKey (default Esc, right-click also cancels)
  • FineAdjustKey (default LeftShift)
  • MoveStep, FineMoveStep, RotateStepDegrees, FineRotateStepDegrees

Optional .vfp wall-face field

  • Use: piece,col,row,type,rotation,wallFace
  • wallFace can be outer or inner.
  • This is used when WallPillarMaterial=Wood so wood wall faces orient correctly.

Preview and Build Controls (Default)

  • Build preview: F8
  • Undo: F9
  • Move preview: UpArrow / DownArrow / LeftArrow / RightArrow
  • Rotate preview: Q / R
  • Confirm build: E
  • Fine adjust: hold LeftShift
  • Cancel preview: Esc or right-click

Terrain Risk Markers

In preview mode, orange risk markers appear for MEDIUM and HIGH terrain edge risk:

  • Terrain-level markers show sampled hotspot locations near problematic edge terrain.
  • Top-edge markers (3 fixed points) appear on the top rim of the green outer wall for better visibility from downhill camera angles.

Risk text reports step and relief metrics to help decide if you should nudge/rotate before building.

What Happens During Build

  1. Terrain snapshot is captured for undo.
  2. Rock-like blockers in the leveled area are cleared.
  3. Terrain is leveled to the highest sampled point.
  4. Terrain physics is polled until collider is ready.
  5. Pieces are spawned at terrain-conforming heights.
  6. Post-build spike guard removes late protruding blockers.

Undo Behavior

  • Removes VFP-tagged pieces within about 75m of player.
  • Restores terrain from the latest captured snapshot in the current session.
  • Terrain visuals may not update immediately after undo; move to another area and come back to refresh chunk visuals.
Piece removal can work across sessions. Terrain restore is session-bound and depends on a valid snapshot captured during the current run.

Tuning Tips

  • Faster test builds: set passes to 1 / 1.
  • Balanced default: 2 / 2.
  • Rough terrain: try 4 / 3 or higher.

Troubleshooting

Build hotkey does nothing

  • Check FloorPlanFile points to a valid file.
  • Check keybind conflicts.
  • Review BepInEx log for load errors.

Placement is wrong

  • Adjust in preview before confirming.
  • Tune BuildOriginForwardOffset.

Pieces float/collapse

  • Let the full build sequence finish.
  • Increase terrain pass settings on difficult ground.

Undo did not visibly restore terrain

  • Undo restores only the latest snapshot captured in the current session.
  • If snapshot capture failed, piece removal may still work while terrain restore does not.
  • Terrain visuals can refresh late; move away from the area and return.
