ValheimFloorPlan - User Guide

Place a pre-made .vfp floor plan in game, preview the footprint, align it precisely, and let the mod level terrain and spawn pieces automatically.

Preview First Auto Terrain Prep One-Key Undo

Quick Start

  1. Set FloorPlanFile to your .vfp file in config.
  2. Enter your world and press F8 to open preview.
  3. Move/rotate preview until placement looks right.
  4. Left-click to confirm build.
  5. Use F9 to undo if needed.

Installation

Thunderstore Mod Manager

  1. Install ValheimFloorPlan in Thunderstore Mod Manager.
  2. Launch the game once.
  3. Close game and update config if needed.

Manual

  1. Install BepInEx for Valheim.
  2. Copy the mod DLL to BepInEx/plugins.
  3. Launch once to generate config.

Configuration

Default config file:

BepInEx/config/com.yourname.valheimfloorplan.cfg

SettingDefaultWhat it does
FloorPlanFileemptyFull path to the .vfp file to build.
BuildHotkeyF8Opens placement preview.
UndoHotkeyF9Removes nearby VFP pieces and restores terrain snapshot.
BuildOriginForwardOffset12Initial preview origin in front of player (10-20).
TerrainLevelPasses2Main terrain leveling pass count (1-5).
TerrainSpikeCleanupPasses2Post-leveling cleanup pass count (1-5).
ExternalWallHeight1Stacks external Wall/Pillar objects to this many levels (1-4).

Preview and Build Controls (Default)

  • Build preview: F8
  • Undo: F9
  • Move preview: UpArrow / DownArrow / LeftArrow / RightArrow
  • Rotate preview: Q / E
  • Fine adjust: hold LeftShift
  • Cancel preview: Esc or right-click
  • Confirm build: left-click

What Happens During Build

  1. Terrain snapshot is captured for undo.
  2. Rock-like blockers in the leveled area are cleared.
  3. Terrain is leveled to the highest sampled point.
  4. Terrain physics is polled until collider is ready.
  5. Pieces are spawned at terrain-conforming heights.
  6. Post-build spike guard removes late protruding blockers.

Undo Behavior

  • Removes VFP-tagged pieces within about 75m of player.
  • Restores terrain from captured snapshot.
Piece removal can work across sessions. Terrain restore depends on a valid captured snapshot from the current run.

Tuning Tips

  • Faster test builds: set passes to 1 / 1.
  • Balanced default: 2 / 2.
  • Rough terrain: try 4 / 3 or higher.

Troubleshooting

Build hotkey does nothing

  • Check FloorPlanFile points to a valid file.
  • Check keybind conflicts.
  • Review BepInEx log for load errors.

Placement is wrong

  • Adjust in preview before confirming.
  • Tune BuildOriginForwardOffset.

Pieces float/collapse

  • Let the full build sequence finish.
  • Increase terrain pass settings on difficult ground.
